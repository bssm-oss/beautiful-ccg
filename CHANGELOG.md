# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/), and this project adheres to [Semantic Versioning](https://semver.org/).

## [0.2.0] - 2026-04-07

### Added

- **Parallel execution** — `parallel` strategy runs all adapters concurrently via `Promise.allSettled`
- **Config loading** — `.ccg/config.yaml` loaded at runtime; routing rules, adapter `enabled` flag honored
- **`bccg pipeline`** CLI command for direct pipeline DSL execution
- **`bccg doctor`** command for health checks (adapters, config, MCP registration)
- **`--json`** flag on `run` and `pipeline` for scripted output
- **`--verbose`** flag shows routing decisions
- **stdin support** — `echo "prompt" | bccg run -`
- **Claude Code plugin** — `.claude-plugin/plugin.json` manifest
- **`.mcp.json`** project-local MCP config generated by `bccg init`
- 8 new tests (133 total)

### Fixed

- Claude Code MCP path: `~/.claude/mcp-servers.json` → `~/.claude.json`
- Version parsing: extract semver from noisy `--version` output
- Copilot model list updated to April 2026 (6 → 18 models, 5 → 10 aliases)
- Added `gpt-5.4`, `gpt-5.4-mini`, `gemini-3.1-pro`, `grok-code-fast-1`, and more
- Removed deprecated `gemini-3-pro`, `claude-sonnet-4`
- Phantom dependency: `adapter-base` added to `cli` and `mcp-server`
- `pnpm lint` fixed (root tsconfig `rootDir` removed, test type casts corrected)

### Changed

- Package scope: `@bccg/*` → `@beautiful-ccg/*`
- CLI package: `beautiful-ccg` → `@beautiful-ccg/cli`
- Router functions no longer accept unused `_config` parameter

## [0.1.0] - 2026-04-01

### Added

- **Monorepo** with 8 packages (`adapter-base`, `adapter-claude`, `adapter-codex`, `adapter-copilot`, `adapter-gemini`, `core`, `mcp-server`, `cli`)
- **4 CLI adapters**: Claude Code, GitHub Copilot, OpenAI Codex, Google Gemini
- **Copilot multi-model routing** with `--model` flag and short aliases (`opus`, `sonnet`, `codex`, `gemini`, `haiku`)
- **Routing strategies**: `cheap-first`, `quality-first`, `balanced`, `parallel`
- **Pipeline DSL** for chaining CLIs (`gemini:summarize -> codex:implement -> claude:review`)
- **Task classifier** (keyword-based: `reasoning`, `coding`, `summarize`, `general`)
- **MCP server** with 3 tools: `bccg_run`, `bccg_pipeline`, `bccg_status`
- **CLI** with 4 commands: `run`, `status`, `init`, `serve`
- **Recursion guard** via `BCCG_DEPTH` environment variable
- **Auto-setup** via `bccg init` (CLI detection + MCP server registration)
- 125 tests across all packages
